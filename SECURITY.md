# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities through public issues.

If the repository host provides a private vulnerability reporting mechanism,
use that channel first. If private reporting is not available, contact the
project maintainer directly through the repository profile and include:

- A clear description of the issue
- Steps to reproduce
- Affected versions or commits
- Any suggested remediation or mitigation

We will aim to acknowledge reports promptly and coordinate a fix before public
disclosure where possible.

## Scope

Security reports are especially relevant for:

- Token handling and storage
- Extension permissions
- Data exposure or unintended network requests
- DOM injection, unsafe rendering, or script injection risks
- Packaging or release integrity issues

## Disclosure Expectations

- Give maintainers reasonable time to investigate and patch
- Avoid publishing proof-of-concept exploits before a fix is available
- Limit public discussion to high-level impact until remediation lands
